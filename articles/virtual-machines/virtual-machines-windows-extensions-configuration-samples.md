# <a name="azure-windows-vm-extension-configuration-samples"></a>Ejemplos de configuración de extensión de máquina virtual de Windows Azure

> [AZURE.SELECTOR]
- [PowerShell - plantilla](virtual-machines-windows-extensions-configuration-samples.md)
- [CLI - plantilla](virtual-machines-linux-extensions-configuration-samples.md)

<br>

Este artículo proporciona la configuración de ejemplo para configurar las extensiones de Azure VM para Windows VM.

Para obtener más información sobre estas extensiones, consulte [Introducción a las extensiones Azure VM.](virtual-machines-windows-extensions-features.md)

Para obtener más información sobre la creación de plantillas de extensión, consulte [creación de plantillas de extensión.](virtual-machines-windows-extensions-authoring-templates.md)

Este artículo enumeran los valores de configuración esperado para algunas de las extensiones de Windows.

## <a name="sample-template-snippet-for-vm-extensions-with-iaas-vms"></a>Fragmento de código de plantilla de ejemplo para las extensiones de VM con máquinas virtuales de IaaS.
Fragmento de código de plantilla para implementar extensiones aspectos como la siguiente:

      {
      "type": "Microsoft.Compute/virtualMachines/extensions",
      "name": "MyExtension",
      "apiVersion": "2015-05-01-preview",
      "location": "[parameters('location')]",
      "dependsOn": ["[concat('Microsoft.Compute/virtualMachines/',parameters('vmName'))]"],
      "properties":
      {
      "publisher": "Publisher Namespace",
      "type": "extension Name",
      "typeHandlerVersion": "extension version",
      "autoUpgradeMinorVersion":true,
      "settings": {
      // Extension specific configuration goes in here.
      }
      }
      }

## <a name="sample-template-snippet-for-vm-extensions-with-vm-scale-sets"></a>Fragmento de código de plantilla de muestra para extensiones de VM con conjuntos de escala de máquina virtual.

    {
     "type":"Microsoft.Compute/virtualMachineScaleSets",
    ....
           "extensionProfile":{
           "extensions":[
             {
               "name":"extension Name",
               "properties":{
                 "publisher":"Publisher Namespace",
                 "type":"extension Name",
                 "typeHandlerVersion":"extension version",
                 "autoUpgradeMinorVersion":true,
                 "settings":{
                 // Extension specific configuration goes in here.
                 }
               }
              }
            }
          }

Antes de implementar la extensión, compruebe la última versión de extensión y reemplace "typeHandlerVersion" con la última versión actual.

El resto de este artículo proporciona ejemplos de configuraciones para las extensiones de máquina virtual de Windows.

Antes de implementar la extensión, compruebe la última versión de extensión y reemplace "typeHandlerVersion" con la última versión actual.

### <a name="customscript-extension-14"></a>Extensión de CustomScript 1.4.
      {
          "publisher": "Microsoft.Compute",
          "type": "CustomScriptExtension",
          "typeHandlerVersion": "1.4",
          "settings": {
              "fileUris": [
                  "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
              ],
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
          },
          "protectedSettings": {
            "storageAccountName": "yourStorageAccountName",
            "storageAccountKey": "yourStorageAccountKey"
          }
      }

#### <a name="parameter-description"></a>Descripción de parámetro:

- fileUris: lista de separados por comas de direcciones URL de los archivos que se descargarán en la máquina virtual por la extensión. Si no se especifica nada, no se descargan archivos. Si los archivos están en el almacenamiento de Azure, la fileURLs pueden marcarse como privado y la correspondiente storageAccountName y storageAccountKey se pueden pasar como parámetros privados para tener acceso a estos archivos.
- commandToExecute: [parámetro obligatorio]: este es el comando que se ejecutará mediante la extensión.
- storageAccountName: [parámetros opcionales]: nombre de la cuenta de almacenamiento para tener acceso a la fileURLs, si están marcados como privados.
- storageAccountKey: [parámetro opcional]: clave de cuenta de almacenamiento para tener acceso a la fileURLs, si están marcados como privados.

### <a name="customscript-extension-17"></a>Extensión de CustomScript 1.7.

Consulte CustomScript versión 1.4 para descripción del parámetro. Versión 1.7 presenta compatibilidad para enviar la secuencia de comandos parameters(commandToExecute) como protectedSettings, en cuyo caso se se cifran antes de enviar. parámetro 'commandToExecute' se puede especificar en la configuración o protectedSettings, pero no en ambos.

        {
            "publisher": "Microsoft.Compute",
            "type": "CustomScriptExtension",
            "typeHandlerVersion": "1.7",
            "settings": {
                "fileUris": [
                    "http: //Yourstorageaccount.blob.core.windows.net/customscriptfiles/start.ps1"
                ],
                "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1"
            },
            "protectedSettings": {
              "commandToExecute": "powershell.exe -ExecutionPolicy Unrestricted -start.ps1",
              "storageAccountName": "yourStorageAccountName",
              "storageAccountKey": "yourStorageAccountKey"
            }
        }

### <a name="dsc-extension"></a>Extensión de DSC.
      {
          "publisher": "Microsoft.Powershell",
          "type": "DSC",
          "typeHandlerVersion": "2.1(Recommendation is to use the latest version)",
          "settings": {
              "ModulesUrl": "https://UrlToZipContainingConfigurationScript.ps1.zip",
              "SasToken": "Optional : SAS Token if ModulesUrl points to Azure Blob Storage",
              "ConfigurationFunction": "ConfigurationScript.ps1\\ConfigurationFunction",
              "Properties": {
                  "ParameterToConfigurationFunction1": "Value1",
                  "ParameterToConfigurationFunction2": "Value2",
                  "ParameterOfTypePSCredential1": {
                      "UserName": "UsernameValue1",
                      "Password": "PrivateSettingsRef:Key1(Value is a reference to a member of the Items object in the protected settings)"
                  },
                  "ParameterOfTypePSCredential2": {
                      "UserName": "UsernameValue2",
                      "Password": "PrivateSettingsRef:Key2"
                  }
              }
          },
          "protectedSettings": {
              "Items": {
                  "Key1": "PasswordValue1",
                  "Key2": "PasswordValue2"
              },
              "DataBlobUri": "optional : https: //UrlToConfigurationData.psd1"
          }
      }

          {
