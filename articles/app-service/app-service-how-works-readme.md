# <a name="how-app-service-works"></a>Cómo funciona la aplicación de servicio

Servicio de aplicaciones de Azure es un servicio de nube diseñado para resolver los problemas prácticos opuestas a ingenieros hoy. Aplicación de servicio se centra en ofrecer productividad superior sin afectar la necesidad de ofrecer aplicaciones a escala de la nube. Aplicación de servicio también proporciona las características y marcos necesarios para crear aplicaciones de línea de negocio de enterprise mientras para los desarrolladores con los lenguajes de desarrollo más populares (. NET, Java, PHP, Node.JS y Python).
Con el servicio de aplicación, los desarrolladores pueden:

* Crear aplicaciones Web de alta escalabilidad
* Crear rápidamente la aplicación móvil fondo con un conjunto de capacidades móviles fácil de usar como fondo de datos, las notificaciones de inserción y de autenticación con aplicaciones móviles. 
* Implementar, implementar y publicar API con la API de aplicaciones.
* Unir aplicaciones empresariales a flujos de trabajo y transformar datos con aplicaciones de lógica.

>[AZURE.INCLUDE [app-service-linux](../../includes/app-service-linux.md)] 

Todos los tipos de aplicación dependen de la plataforma de aplicación Web scalable y flexible que permite a los desarrolladores una experiencia optimizada ciclo de vida de diseño de la aplicación al mantenimiento de la aplicación. Permitir que las capacidades de ciclo de vida para:

* Creación de aplicación rápida: empezar desde cero o seleccione un paquete de sistemas operativos de Azure Marketplace. 
* Implementación continua: implementar automáticamente el nuevo código de soluciones de control de origen populares como TFS, GitHub y BitBucket y sincronizar el contenido de los servicios de almacenamiento en línea como OneDrive y DropBox.
* Probar en producción: sin problemas entornos de producción antes de crear y administrar la parte del tráfico que se va a ellos. Depurar en la nube cuando sea necesario y deshacer si se encuentran problemas.
* Tareas asincrónicas en ejecución y procesos - ejecutar el código de un proceso de fondo o activar el código en función de eventos (como mensajes de inicio de una cola de almacenamiento de Azure) y horas (CRON) programadas.
* Ajuste de escala de la aplicación: usar una de muchas opciones para ajustar su servicio horizontal y vertical automáticamente en función de tráfico y utilización de recursos. Configurar los entornos privados dedicados a las aplicaciones   
* Mantenimiento de la aplicación: aproveche muchos de los diagnósticos y de depuración características Adelántese problemas y eficaz resolverlos en tiempo real (con características como resolución de problemas y depuración live) o después del hecho mediante el análisis de registros y memoria vuelca
 
Colocar juntos, las capacidades de servicio de aplicación permiten a los desarrolladores centrarse en su código y llegar a un estado de producción estable, scalable muy rápidamente. Con las características de la API y lógica de aplicación, los desarrolladores pueden crear aplicaciones empresariales reales de puente obstáculos entre soluciones empresariales, así como de local a la integración de nube.
